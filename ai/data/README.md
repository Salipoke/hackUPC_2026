## Schema

```
temperature,humidity,wind,light,airQuality,label
```

- `temperature` — Celsius
- `humidity`    — % relative
- `wind`        — km/h
- `light`       — lux
- `airQuality`  — 0..100 index (higher = worse)
- `label`       — `low` or `high`

## Regenerate

```bash
# Pure synthetic, default 7000 rows
npm run dataset

# Mix synthetic + real Open-Meteo Barcelona records (recommended)
npm run dataset:real

# Reproducible run
BIOMESH_SEED=42 npm run dataset:real

# Custom row count
BIOMESH_ROWS=10000 npm run dataset:real
```

Output goes to `ai/data/biomesh-train.csv`. The script prints a summary with row count, class balance and sha256 hash.

## Labelling rule

A row is labelled `high` if any of the following hold:

```
temperature >= 35 °C
temperature <= 0  °C
wind        >= 40 km/h
airQuality  >= 75
(temperature >= 30 °C  AND  humidity <= 25 %)
```

Otherwise `low`. The compound rule is what justifies an ML model over a pure threshold check: the AND/OR/IF tree is non-linear in feature space, and an MLP captures it cleanly while generalising to nearby ambiguous cases.

## Sources

### Synthetic (always on)

Gaussian sampling around six regimes of Barcelona conditions:

| Regime          | Probability | Description |
|-----------------|-------------|-------------|
| Normal day      | 60 %        | Mild temps, average humidity/wind |
| Summer warm day | 20 %        | Warm but safe, slightly elevated AQI |
| Heatwave        | 10 %        | T ≥ 35 °C, low humidity, poor AQI |
| Windstorm       |  5 %        | Strong wind, otherwise mild |
| Cold snap       |  3 %        | Sub-zero temps |
| Pollution event |  2 %        | High AQI, calm wind |

`light` correlates with temperature as a proxy for solar radiation.

### Open-Meteo (when `BIOMESH_FETCH_OPENMETEO=1`)

Free historical hourly archive at `https://archive-api.open-meteo.com`. License: CC BY 4.0. No API key required.

Pulls 4 windows for Barcelona (lat 41.39, lon 2.16):

- 2023-07-15 .. 2023-07-31 (summer heat)
- 2024-02-01 .. 2024-02-14 (winter)
- 2024-08-01 .. 2024-08-15 (summer)
- 2025-01-10 .. 2025-01-24 (winter)

Variables: `temperature_2m`, `relative_humidity_2m`, `wind_speed_10m`, `shortwave_radiation`. Light is derived from shortwave radiation (1 W/m² ≈ 120 lux).

`airQuality` is **not** in the free Open-Meteo archive without a key, so it is synthesised with a temperature-and-wind biased correlation.

## Class balance

Typical output: ~85 % `low`, ~15 % `high`. This reflects real-world distribution. Compensate during training via:

- **Edge Impulse**: enable "Balance dataset" in the impulse settings, or
- **Manual**: pass `class_weight='balanced'` to the Keras compile step.

If you need a balanced CSV for testing, run:

```bash
BIOMESH_ROWS=20000 npm run dataset:real
# then downsample 'low' rows externally
```
